# 패치노트 자동 작성 스킬

이 프로젝트는 롤 내전 디스코드 봇입니다. 기능 추가/수정 작업이 완료되면 반드시 아래 절차를 따르세요.

## 트리거 조건

- 사용자가 기능 추가, 수정, 버그 수정 등 코드 변경 작업을 완료했을 때
- 사용자가 "패치", "배포", "완료" 등의 키워드를 사용했을 때

## 수행 절차

### 1. package.json 버전 업데이트

- 기능 추가: minor 버전 올림 (예: 1.1.0 → 1.2.0)
- 버그 수정: patch 버전 올림 (예: 1.1.0 → 1.1.1)

### 2. src/utils/patchnotes.js 업데이트

`PATCH_NOTES` 객체에 새 버전 항목을 추가합니다:

```js
'버전': {
  date: 'YYYY-MM-DD',  // 현재 날짜
  title: '패치 제목',    // 변경 내용을 한 줄로 요약
  changes: [
    '변경사항 1 — 구체적으로 작성',
    '변경사항 2',
  ],
},
```

작성 규칙:
- 한국어로 작성
- 명령어는 백틱으로 감싸기 (예: \`!명령어\`)
- 사용자 관점에서 어떤 게 바뀌었는지 작성 (코드 내부 변경은 생략)
- 변경사항은 간결하게, 한 줄에 하나씩

### 3. README.md 패치 내역 업데이트

`## 패치 내역` 섹션에 새 항목을 추가합니다. 가장 최근 패치가 위에 오도록 작성합니다.

## 주의사항

- 기존 PATCH_NOTES 항목은 절대 수정하지 않습니다
- 버전 번호는 semver를 따릅니다
- 패치노트 작성 후 사용자에게 변경된 버전과 내용을 알려줍니다
